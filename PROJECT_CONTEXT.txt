App Entry And Auth Routing (Api-integration)
- SplashScreen: SplashNavigationResolver checks token + has_seen_onboarding.
- Valid token -> mainShell (dashboard).
- No token + first launch -> OnboardingScreen.
- No token + seen onboarding -> LoginScreen.
- Login success -> mainShell.
- Register success -> addVehicle (OTP skipped).
- Remember Me: saves email+password; NOT cleared on logout.
- Logout: clears token; optional POST auth/logout when live.
- Add vehicle: POST vehicles per plate (form-urlencoded).
- Main shell: UserProfileProvider loads GET auth/me.

Home Station API (done)
- All: GET service-stations.
- Nearby: GET service-stations/nearest.
- Less distance: GET suggest-nearest (auth + POST locations first).
- Station cards include API id for detail/booking.

Profile & Vehicles API
- GET auth/me, POST vehicles (one plate per request).
