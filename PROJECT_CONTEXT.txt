App Entry And Auth Routing (Api-integration)
- SplashScreen: no fixed delay; SplashNavigationResolver checks token + has_seen_onboarding.
- Valid token -> mainShell (dashboard).
- No token + first launch -> OnboardingScreen (sets has_seen_onboarding when completed).
- No token + seen onboarding -> LoginScreen.
- Login success -> mainShell.
- Register success -> addVehicle (OTP flow skipped for now).
- Remember Me: saves email+password in LocalStorageService when checked; prefilled on return; NOT cleared on logout.
- Logout: AuthRepository.logout() clears token only; onboarding + remember-me persist; restart goes to login not onboarding.

Home Station API (Api-integration)
- Tabs: All | Nearby | Less distance (single row, filter, no overflow).
- All: GET service-stations.
- Nearby: GET service-stations/nearest (lat, lng, radius).
- Less distance: GET suggest-nearest (NOT service-stations/suggest-nearest — 404).
- suggest-nearest requires auth + POST locations with user coordinates before fetch.
- Response data.suggested_stations[] via ServiceStationMapper.
- HomeProvider.loadStations() uses try/finally so loading always clears.
